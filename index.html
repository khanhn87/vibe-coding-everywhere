Blueprint

Vibe Coding Everywhere
from Telegram

A practical setup guide for running your coding workflow remotely with Codex CLI, CCPoke, VPS, and GitHub.

Start Setup

System Stack

Telegram Codex CLI CCPoke Ubuntu VPS GitHub

Telegram sends instructions. CCPoke receives and orchestrates. Codex CLI executes coding tasks on your VPS. GitHub stores code, history, and CI.

Prerequisites

  • A Linux VPS with SSH access
  • A GitHub account and repo
  • Telegram account + bot token
  • Codex CLI installed and authenticated
  • CCPoke installed/configured on VPS

Setup Flow

  1. 1. Prepare VPS

    Install runtime dependencies and secure SSH/firewall first.

    sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl build-essential

  2. 2. Connect GitHub

    Choose one method below (HTTPS or SSH), then verify remote access from server.

    # Option A: HTTPS
git clone https://github.com/your-org/your-repo.git
cd your-repo
git remote -v

# Option B: SSH
git clone git@github.com:your-org/your-repo.git
cd your-repo
git remote -v

  3. 3. Configure Codex CLI

    Authenticate Codex CLI and confirm command execution works.

    codex --version
codex auth login
codex run "echo ready"

  4. 4. Wire CCPoke + Telegram

    Set your bot token/webhook and map incoming messages to your execution workflow.

    export TELEGRAM_BOT_TOKEN=your_token
export TELEGRAM_CHAT_ID=your_chat
ccpoke start

  5. 5. Run End-to-End Test

    Send a Telegram command and verify VPS execution + GitHub update.

    # Telegram message example
/deploy main

# Verify on VPS
git log --oneline -n 3

Production Notes

  • Run services with `systemd` or PM2 for auto-restart.
  • Use GitHub Deploy Keys with least privilege.
  • Store secrets in `.env` and never commit them.
  • Add a command allowlist to avoid arbitrary execution from Telegram.
